# 📋 QA Samples — Примеры тестовой документации

В этом репозитории собраны примеры моих артефактов, которые я создал в процессе тестирования.  
Здесь вы найдете:

- 🐞 **Баг-репорты** — реальные примеры оформления дефектов (структурированно, с шагами и скриншотами)
- ✅ **Чек-листы** — покрытие функциональности для быстрой проверки
- 📝 **Тест-кейсы** — детальные сценарии с ожидаемыми результатами

## 📂 Структура

| Папка | Содержание |
|-------|------------|
| [bug-reports](https://github.com/Midas769/QA-samples/tree/main/bug-reports) | Баг-репорты |
| [checklists](https://github.com/Midas769/QA-samples/tree/main/checklists) | Чек-листы для регрессионного тестирования |
| [test-cases](https://github.com/Midas769/QA-samples/tree/main/test-cases) | Подробные тест-кейсы с предварительными условиями и шагами |


## 🎯 Цель этого репозитория

Показать мой подход к тестированию:
- **Системность** — покрытие всех важных сценариев
- **Детализацию** — каждый баг описан так, что разработчик поймет без лишних вопросов
- **Качество** — четкий язык, отсутствие воды, приоритизация


## 🖼 Примеры скриншотов

Все скриншоты к баг-репортам хранятся в папке `bug-reports/screenshots/` и вставлены в соответствующие отчеты.
